Prototipos de Mapa de Recorridos

Representación de Senderos, Intersecciones y Puntos de Referencia por medio de Grafos.

Este desarrollo forma parte del Proyecto Final de Grado de la carrera Ingeniería Informática de la Facultad de Ingenieria - UNJu.

Aquí se muestra un listado de los diferentes prototipos, cada uno con su descripción.

Prototipo 1

Utilización de la Libreria VisJs con AngularJs para crear grafo a partir de un panel de control que permite la Generación de Nodos y Arcos.

Prototipo 2

El Prototipo 1 mostro la dificultad que se tiene al crear un grafo que represente el recorrido creando de forma individual los arcos y nodos, es por ello que se replanteo la forma de generación de un mapa de recorridos a traves de la generación de trayectos. Cada trayecto que se crea, es una sucesion de nodos conectados por arcos.

Además se estableció una serie de restricciones al momento de crear una trayecto:

  • No existe una arista que se conecte a un mismo nodo.
  • Entre dos nodos diferentes solo existe una arista que los une.

Prototipo 3

En este prototipo se planteó mejorar los controles de usuario para la creación de trayectos.

Los controles que se agregan son:

  • Restablecer el formulario de trayecto.
  • Limpieza de campos del formulario para agregar nueva trayecto.
  • Creación rapida de N campos que representan la sucesion de nodos conectados.
  • Eliminación de Nodos y Arcos.

Prototipo 5 El objetivo en esta version del prototipo es implementar un algoritmo de la ruta mas corta entre dos nodos.
Se agregaron las siguientes funciones al prototipo:
  • El usuario puede establecer el valor de la distancia que existe entre dos nodos, para ello selecciona un arco del grafo y luego establece el valor de la distancia.
  • El usuario puede cambiar el tipo de nodo a Intersección o Punto de Referencia, de forma predeterminada cuando un nodo es creado tiene el valor de Intersección.
  • Se agrego libreria para calcular el camino más corto mediante el algoritmo de Dijkstra.

Prototipo 6 El objetivo en esta version del prototipo es implementar la información referencial segun la dirección que tenga un nodo hacia sus nodos vecinos. La cantidad maxima de conexiones en un nodo intersección hacia otros es de 4, las cuales estaran orientadas segun Izquierda, Derecha, Adelante, Atras.
Se agregaron las siguientes funciones al prototipo:
  • Interfaz para el ingreso de información sobre las direcciones que existen desde un nodo.
  • Se agrego la funcion para almacenar las direciones hacia cada nodo.
